### 技术选型（已锁定）
- **架构**：简单 Python Pipeline + 条件退出（非 LangGraph、非 CrewAI）
- **理由**：流程是单向价值传递，带条件退出，不是真正的 Graph，用 LangGraph 反而复杂化
- **模型**：Claude claude-sonnet-4-6
- **人机协作层**：飞书机器人（OpenClaw + 飞书消息作为 Human-in-the-loop 界面）
- **展示层**：静态网页

### 4D 原则的落法
- **Delegation**：AI 并行写 6 个 Agent Prompt、Pipeline 代码、Demo 数据、展示页；Jacky 做判断和确认
- **Description**：每次委托必须给清：角色定义 + 输入格式 + 输出格式
- **Discernment**：Agent 划分方式、Demo 触发场景、技术栈选择——由 Jacky 拍板
- **Diligence**：每个 Agent 单独测试后再接入 Pipeline，不一次性全跑

### 工程亮点方向（已明确）
1. **JSON Schema 强校验 + Structured Output**：Agent 间数据契约，防止链路断裂
2. **守门 Agent 判断 rubric + few-shot examples**：让判断稳定、可解释、可审计
3. **Human-in-the-loop 设计**：关键节点（守门完、发版评审完）暂停等待人工确认，通过飞书推送

### 价值定位（已明确）
- 不是替代人的判断，而是**把"应该做"变成"自动发生"**
- 把"做正确的事的成本"降到接近零，让流程真正跑起来
- 标准来源：2026-04-27 会议纪要——会议里定了但执行不下去的那套流程，现在工程化
